引言

二分查找

O(log2 n)

大O表示法

  • O(log2 n) 对数时间 二分查找
  • O(n) 线性时间 简单查找
  • O(n * log2 n) 快排
  • O(n^2) 选择排序
  • O(n!) 旅行商问题

    小结

  • 二分查找比简单查找快的多
  • O(log2 n) 比O(n)快
  • 算法运行时间不已秒为单位
  • 算法运行时间是从其增速角度度量的
  • 算法运行时间用大O表示法表示

选择排序

内存工作原理

数组和链表

选择排序

  • O(n^2)

    小结

  • 计算机内存犹如一大堆抽屉
  • 存储多个元素时, 可选择数组或链表
  • 所有的元素都在一起
  • 链表的元素是分开的, 每个元素存储了下一个元素的地址
  • 数组的读取速度很快
  • 链表的插入和删除速度很快
  • 同一个数组中, 所有元素类型必须相同

递归

递归

基线条件和递归条件

每个递归函数都有两部分: 基线条件(函数不再调用自己)和递归条件(函数调用自己)

栈

调用另一个函数时, 当前函数暂停并处于未完成状态. 该函数的所有变量的值都还在内存中

  • 递归栈 存储详尽的信息需要大量内存, 每个函数调用都要占用一定的内存.
    • 重新编写代码 转而使用循环
    • 使用尾递归

小结

  • 递归指的是调用自己的函数
  • 每个递归函数都要两个条件 基线条件和递归条件
  • 栈的两种操作: 压入和弹出
  • 所有函数调用都进入调用栈
  • 调用栈可能很长这将占用大量内存

快速排序

分而治之D&C

快速排序

再谈大O表示法

  • 平均情况
  • 最糟情况

    快速排序, 平均(最佳)情况 有O(log2 n)层, 最差情况有O(n)层, 每一层均需处理n个元素, O(n) 即快排最差情况O(n^2), 平均情况O(log2n)

小结

  • D&C将问题逐步分解. 处理问题时, 基线条件很可能是空数组或只包含1个元素的数组
  • 实现快速排序, 随机选用基准值元素, 平均时间为O(n * log2 n)
  • 大O表示法中常量有时候事关重大, e.g. 快速排序O(n * log2 n)快于合并排序O(n * log2 n)
  • 比较简单查找和二分查找时, 常量无关, 因为列表很长时O(log2 n) 比 O(n)快得多

散列表

散列函数

  • 一致性
  • 不同输入映射到不同数字

    包含额外逻辑的数据结构

应用案例

  • 小结(适用于)
    • 仿真映射关系
    • 防止重复(每人仅投一票例子)
    • 缓存

冲突

不同输入映射到同输出情况, 常见解决方式是输出位置保存链表

  • 散列函数很重要, 理想情况是,将键均匀映射到散列表不同位置
  • 散列表存储的链表很长, 散列表速度将急剧下降(散列函数好, 链表不会很长)

性能

  • 避免冲突
    • 较低的填装因子(数组中被占用的位置数)
    • 较好的散列函数

小结

  • 结合散列函数 和 数组 创建散列表
  • 使用最大限度减少冲突的散列函数
  • 查, 插, 删速度都非常快
  • 散列表适用于仿真映射
  • 装填因子大于0.7, 调整散列表长度
  • 用于缓存
  • 适合用于防止重复

广度优先搜索

图简介

图是什么

广度优先搜索

找到的不仅是A到B的路径, 而且是最短路径

  • 队列

实现图

散列表实现图(由多个节点, 且与临近节点相连)

  • 有向图 & 无向图

实现算法

  • 运行时间O(V+E) 人数+边数
  • 拓扑排序(列表有序), 根据图创建有序列表
  • 树

小结

  • 广度优先搜索指出是否有A到B的路径
  • 广度优先搜索指出A到B的最短路径(若有)
  • 面临类似寻找最短路径问题, 使用图创建模型, 再用广度优先搜索解决问题
  • 有向图边为箭头, 指定关系的方向
  • 无向图边不带箭头, 关系双向
  • 队列FIFO
  • 栈LIFO
  • 需要按加入顺序检查搜索列表的人, 否则找到的不是最短路径, 故使用队列
  • 经过检查的人不再次检查, 否则导致无限循环

Dijkstra

使用狄克斯特拉算法

术语

  • 权重
  • 只适用于DAG 有向无环图

    换钢琴

  • 找出开销最低节点, 并更新其邻居开销

    负权边

  • 不可使用Dijkstra, 原因: 已处理过的节点有可能被更新开销

    实现

  • Graph{Hash Table}
  • Cost{Hash Table}
  • Parent{Hash Table}
  • Processed[]

小结

  • 广度优先搜索用于非加权图查找最短路径
  • 狄克斯特拉 用于加权图中查找最短路径
  • 仅当权重为正, Dijkstra才管用, 否则使用贝尔曼-福特算法

贪婪算法

教室调度问题

每步选择局部最优解, 最终得到的就是全局最优解

背包问题

贪婪策略不能获得最优解, 但非常接近

集合覆盖问题

NP完全问题

  • e.g. 旅行商问题
  • 识别
    • 元素少运行速度快, 随着元素增加, 速度慢
    • 涉及所有组合的问题, 通常是NP完全问题
    • 不能将问题分成小问题, 必须考虑各种可能情况, 可能是NP完全问题
    • 涉及序列且难以解决, 可能是NP完全问题
    • 涉及集合且难以解决, 可能是NP完全问题
    • 问题可转换成集合问题或旅行商问题, 肯定是NP完全问题

小结

  • 贪婪算法寻找局部最优解, 企图以这种方式获得全部最优解
  • NP完全问题没有找到快速解决方案
  • 面临NP完全问题, 最佳做法为近似算法
  • 贪婪算法易于实现, 运算速度快, 是不错的近似算法

动态规划

背包问题

  • 比较更大价值
    • Cell[i-1][j]
    • 当前商品价值 + Cell[i-1][j-当前商品重量]

背包问题FAQ

  • 在增加一种样品
  • 行的顺序变化, 结果不变化
  • 仅当每个问题离散, 动态规划才管用, 不然没法建模

最长公共子串

  • 动态规划在给定的约束条件下找到最优解
  • 可被分解独立且离散的子问题, 可使用动态规划解决
  • 每种动态规划都涉及网格
  • 单元格中的值, 即是要优化的值
  • 每个单元格都是子问题, 考虑将问题分成子问题, 有助于找出网格坐标轴
  • e.g 最长公共子串(hish vs fish or vista) 最长公共子序列 (fosh vs fort or fish )
